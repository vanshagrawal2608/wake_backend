## What Wake is
An **adaptive AI alarm** for iOS (SwiftUI). You set *when you must be out of bed*; it builds a gentle→loud "sunrise" wake plan and learns your wake-up over time. You confirm you're awake by **saying "I'm awake"** — judged smartly, no voice enrollment.

## The architecture decisions (don't re-litigate)
- **No voice enrollment / baseline.** Judge awakeness from clarity of "I'm awake" on-device; escalate to Gemini only when unsure. (`WakeJudge.swift`, `VoiceWakeVerifier.swift`)
- **Cloud = Gemini 2.5 Flash, called DIRECTLY from the app** (`GeminiDirectJudge.swift`) — no backend server. Key lives in gitignored `Secrets.xcconfig` (`WAKE_GEMINI_KEY`), protected by a budget cap. The `backend/` folder is an OPTIONAL proxy kept only for a future public release.
- **Distribution: free personal signing / SideStore**, personal + a few shared devices. No App Store, no $99.
- **Palette:** ember / blood-red / maroon, gradient-forward, committed dark "night" world.
- Design source of truth = the mockup at `mockup/index.html` (also a published Artifact).

## Known gotchas / environment
- **Device signing fails on office/Zomato network** — "Communication with Apple failed." Fix: turn off VPN, use a **personal hotspot / home Wi-Fi**, then *Try Again* in Signing & Capabilities. Bundle id set to `com.vansh.wake`, Team = Personal.
- **Voice on Simulator** needs: Mac online (uses Apple **server** recognition there — error 1101 on local is why), macOS mic permission for Simulator, and the in-app Mic+Speech prompts allowed. On real device it's fully on-device.
- **Free signing = 7-day expiry**; re-run ⌘R weekly (or SideStore auto-refresh).
- Can't `git push` from the Claude sandbox reliably (network) — user pushes from their own terminal; `gh` login is set up.
